# un conflit git est arrivé parce qu'on a créé un fichier localement qui porte le même nom qu'un fichier déjà présent sur la branche de destination, et Git refuse de l'écraser pour éviter qu'on perde notre travail.
# La méthode consiste à mettre notre travail local en sécurité (via un git stash ou un commit), ou à supprimer le fichier en conflit, afin de libérer le chemin pour que Git puisse changer de branche sans écraser nos modifications.
